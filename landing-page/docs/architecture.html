02 · Host-aware routingmiddleware.ts decision tree

One middleware, three hosts. Apex always redirects to www. www-side admin/sign-in/auth paths redirect to the portal subdomain so the auth cookie lives on a single host (no double-sign-in). Portal-side passthroughs render literal paths; everything else rewrites under /portal/*. 

flowchart TD
    req{Request host}

    req -->|easytradesetup.com| apex_redir["301 → www"]

    req -->|www.easytradesetup.com| www_path{path}
    www_path -->|/admin/*| www_admin["301 → portal/admin"]
    www_path -->|/sign-in /sign-up
/auth/*| www_auth["301 → portal"]
    www_path -->|/portal/*| www_portal["301 → portal cleaned"]
    www_path -->|else| render_mkt["render /(marketing)/*"]

    req -->|portal.easytradesetup.com| portal_path{path}
    portal_path -->|/portal/*| portal_strip["301 → drop /portal prefix"]
    portal_path -->|/admin /sign-* /auth
/api /_next /favicon| pass[passthrough]
    portal_path -->|else| rewrite["rewrite to /portal/*"]

    pass --> auth_check{needsAuth?}
    rewrite --> auth_check
    auth_check -->|no| render["render"]
    auth_check -->|yes, no session| signin_redir["302 → /sign-in"]
    auth_check -->|yes, has session| admin_check{path = /admin/*?}
    admin_check -->|no| render
    admin_check -->|yes, in admins| render
    admin_check -->|yes, NOT in admins| four04["notFound"]

04 · Data modelPostgres schema · 10 tables

User identity owned by auth.users (Supabase Auth, UUID id). Application tables key on user_id text for compatibility with prior Clerk shape — cast at join boundaries. RLS enabled on every app table; service-role bypasses for server-side queries only.

05 · Authentication flowThree sign-in methods, one cookie host

All sign-in paths target portal.easytradesetup.com so the Supabase session cookie lives on a single host. www-side /sign-in 301s to portal before form render. 

sequenceDiagram
    actor U as User
    participant W as www / portal page
    participant SI as /sign-in
    participant SB as Supabase Auth
    participant G as Google OAuth
    participant CB as /auth/callback
    participant P as /portal or /admin

    U->>W: click Sign in
    W->>SI: 301 if on www

    alt Google OAuth
        U->>SI: click Continue with Google
        SI->>G: signInWithOAuth
        G-->>U: consent screen
        U->>G: approve
        G->>CB: redirect with code
        CB->>SB: exchangeCodeForSession
        SB-->>CB: session cookie
        CB->>P: 302 → /portal
    else Magic link
        U->>SI: email
        SI->>SB: signInWithOtp
        SB-->>U: email link
        U->>CB: click link
        CB->>SB: exchange
        SB-->>CB: cookie
        CB->>P: 302 → /portal
    else Password
        U->>SI: email + password
        SI->>SB: signInWithPassword
        SB-->>SI: cookie
        SI->>P: window.location = redirect
    end

    P->>P: middleware checks session
+ admins row if /admin/*

06 · Pageview trackingPrivacy-respecting self-hosted analytics

No raw IP stored. visitor_id = SHA-256 of (salt + IP + UA + day) — rotates daily so no cross-day re-identification. Bot UA regex drops crawlers. Skip-list excludes /api, /portal, /admin, /sign-*, /auth. 

flowchart LR
    visit["Marketing page view"] --> tracker["PageviewTracker
client component"]
    tracker -->|POST /api/track/pageview| api["Node route handler"]

    api --> bot{UA matches bot regex?}
    bot -->|yes| skip1[skip]
    bot -->|no| skip_path{path in skip-list?}

    skip_path -->|yes| skip2[skip]
    skip_path -->|no| hash["sha256(salt + ip + ua + day)
→ 32-char visitor_id"]

    hash --> insert[("INSERT INTO pageviews")]
    insert --> admin["Admin reads last 30d
uniques + top-5 paths"]
